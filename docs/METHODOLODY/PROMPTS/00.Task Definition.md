[Твоё нечёткое описание. Например: "мне нужно чтобы таблица Route 
также имела статусы аналогично как это есть в Заказах"]

Задание:
1. **Уточни требования** через вопросы:
   - Что конкретно нужно реализовать?
   - Какие use cases?
   - Кто будет использовать?
   - Какие ограничения?

2. **Найди аналоги/бенчмарки** в проекте:
   - Похожие реализации в текущей кодовой базе
   - Паттерны которые можно переиспользовать
   - Модули для reference

3. **Сформулируй задачу чётко**:
   - Название задачи (кратко, понятно)
   - Описание (что нужно сделать)
   - Use cases (кто и как будет использовать)
   - Scope (что входит, что НЕ входит)
   - Success criteria (когда задача считается выполненной)

4. **Определи сложность и оценку**:
   - Complexity: Low / Medium / High
   - Estimated effort: X hours
   - Risk factors

5. **Запиши в CURRENT_TASK.md** в секцию ## Task Definition
